# Non-Functional Requirements

这份文档定义 DataPilot 的底座要求。它们不是第一眼可见的功能，但会决定产品是否能长期稳定、轻量、安全、好用。

## 1. 数据存储

### 桌面端

桌面端优先使用本地 SQLite 作为元数据存储。

保存内容：

- 数据源配置，不包含明文密码。
- 查询历史。
- 保存查询。
- 工作区 Tab。
- 编辑器草稿。
- 用户偏好。
- 本机审计缓存。
- AI 会话索引和摘要。

要求：

- 存储路径必须可预期，遵循系统应用数据目录。
- 支持导入和导出配置。
- 支持 schema migration。
- 不允许把数据库密码、SSH 私钥、AI Key 明文写入 SQLite。

### Web / Server 模式

Web 服务端第一阶段也可以使用 SQLite，便于轻量部署。后续需要支持 PostgreSQL 作为企业部署元数据库。

保存内容：

- 用户、角色、权限。
- 团队和工作区。
- 数据源配置。
- 查询资产。
- 审计日志。
- 策略配置。
- AI Provider 配置。

## 2. 密钥和凭证管理

敏感信息必须独立于普通配置存储。

### 桌面端

优先使用系统安全存储：

- macOS：Keychain。
- Windows：Credential Manager。
- Linux：Secret Service / libsecret。

敏感信息：

- 数据库密码。
- SSH 私钥和 passphrase。
- AI Provider API Key。
- 代理认证信息。

如果系统安全存储不可用，必须提示用户并使用本地加密文件兜底，不能静默降级为明文。

### Web / Server 模式

Web 模式必须支持：

- 服务端加密存储。
- 主密钥通过环境变量或外部 secret 注入。
- 密钥轮换。
- 权限控制。
- 审计访问记录。

## 3. 插件和驱动管理

数据库支持必须插件化，避免核心层被方言和驱动污染。

每个数据库插件需要声明：

- 数据库类型。
- 支持版本。
- 驱动依赖。
- 连接参数 schema。
- 元数据能力。
- SQL 方言能力。
- 查询计划能力。
- 导入导出能力。
- 安全能力限制。

驱动管理要求：

- 支持查看已安装驱动。
- 支持驱动版本。
- 支持禁用插件。
- 支持插件能力探测。
- 生产环境禁止自动下载不可信驱动。

第一阶段可以内置 MySQL、PostgreSQL、SQLite 驱动，插件市场后置。

## 4. 同步、备份和迁移

桌面优先不代表数据只能留在本机。配置资产必须可迁移。

第一阶段：

- 导出连接配置，不包含密钥明文。
- 导出查询历史和保存查询。
- 导出用户偏好。
- 导入配置包。

后续阶段：

- 团队工作区同步。
- 跨设备同步。
- Git-friendly 查询资产导出。
- 审计日志归档。
- 配置备份加密。

## 5. 性能和体验指标

产品要比 Navicat 更顺手，必须有可测量指标。

### 桌面端指标

- 冷启动目标：小于 2 秒进入可交互界面。
- 常驻内存目标：空工作区小于 200 MB。
- 应用包体：显著低于 Electron 同类方案。
- 常用操作必须支持快捷键。
- Command Palette 打开延迟小于 100 ms。

### 查询和结果集指标

- 10 万行结果集必须保持滚动可用。
- 大结果集默认分页或流式加载。
- SQL 执行必须显示耗时、行数、限制和截断状态。
- 长查询必须可取消。
- 导出任务必须异步执行并可查看进度。

### 安全指标

- AI 生成 SQL 默认不直接执行。
- 高风险 DDL/DML 必须二次确认。
- 生产环境默认启用更严格策略。
- 无 `WHERE` 的 `UPDATE` / `DELETE` 必须阻断或强提醒。
- 所有导出和危险操作必须可审计。

### 可用性指标

- 常用查询路径点击数少于 Navicat。
- 新建连接到执行第一条 SQL 的路径要尽量短。
- 最近连接、最近查询和最近对象必须可快速打开。
- 工作区崩溃或退出后可恢复草稿和 Tab。

## 6. 可观测性

系统需要从第一阶段保留诊断能力：

- API 请求日志。
- SQL 执行日志。
- 慢查询记录。
- AI 请求耗时。
- 错误日志。
- 桌面端本地诊断包导出。

诊断日志不得包含明文密码、API Key 或完整敏感数据结果。

## 7. 升级和兼容

- 本地 SQLite schema 必须可迁移。
- 配置文件必须带版本号。
- 插件能力声明必须可向后兼容。
- 桌面端升级不能丢失连接、历史和偏好。
- 重大迁移前必须备份本地元数据。
